# 花园重构：整体页面+9地块

## 概述

当前花园采用分布式切换的插槽管理方式（5个插槽），每次只能查看一个插槽，体验碎片化且缺乏沉浸感。将其重构为整体花园页面，通过划分9个地块（3×3 网格，1免费+8阶梯解锁）实现一页管理多块种植区域，视觉风格自然清新，提升种植体验的整体感和愉悦感。

## 需求

- [x] 将花园从分布式插槽切换改为整体页面布局
- [x] 设计并实现 9 个地块（plots）的网格布局，视觉风格自然清新
- [x] 地块解锁梯度修改为：`[100, 100, 1000, 1000, 2000, 2000, 3000, 3000, 5000, 5000]`（原为 `[100, 1000, 5000, 10000, 30000]`）
- [x] 地块 0 免费，地块 1-10 按上述梯度依次解锁
- [x] 保留现有植物种类和生长机制（fruit_tree, care_tree, gift_tree, money_tree, golden_apple_tree）
- [x] 每个地块上可独立种植/浇水/施肥/收获，逻辑复用现有 garden.ts 机制
- [ ] 探索更优美的植物/树木贴图方案（现有贴图作为后备）
- [x] 地块状态（empty/growing/ready/withered）有清晰的视觉区分
- [x] 添加地块切换/操作的平滑动画过渡
- [x] 更新 i18n 中英文文案以匹配新布局

## 非目标

- 不修改树木生长周期、掉落池、价格等核心数值（仅改地块数量和解锁成本）
- 不修改工具升级系统（watering_can, shovel, fertilizer_box）
- 不改变花园与成就、伙伴日程等外部系统的交互接口

## 技术方案

- **核心逻辑**：`src/core/garden.ts` — 修改 `gardenSlotCount` 和 `gardenSlotUnlockCosts`，扩展类型以支持更多地块；添加数据迁移逻辑
- **UI 组件**：`src/ui/GardenPage.tsx` — 从单插槽切换式重写为整体网格布局；每个地块卡片展示树生长状态、操作按钮
- **CSS 样式**：`src/styles/garden.css` — 重写样式为自然清新风格（大地色系、圆角卡片、柔和阴影、植物装饰元素）
- **控制器**：`src/ui/app/useGardenController.ts` — 适配新地块数量
- **贴图资源**：`src/assets.ts` — 更新树阶段贴图引入，寻找更优美的植物贴图素材
- **类型定义**：`src/core/petTypes.ts` — 扩展 `GardenSlot`/`GardenState` 以支持最多 11 个地块
- **数据迁移**：`src/core/saveCodec.ts` — 处理旧存档（5槽）到新存档（11槽）的迁移

## 验收标准

1. 花园页面展示 11 个地块（1 个免费 + 10 个阶梯解锁），网格布局
2. 地块解锁成本符合新梯度
3. 在地块上可完成种植、浇水、施肥、收获完整流程
4. 地块状态（empty/growing/ready/withered）通过视觉元素清晰可辨
5. 整体视觉风格自然清新（大地色系、植物装饰、柔和动画）
6. 地块切换/操作有平滑动画
7. 现有存档数据迁移后兼容新地块结构
8. 构建通过（`npm run build`）
