# PocPet 打包发布流程

本文记录当前 Windows 开发机上的标准测试打包流程。除非特别说明，命令都在仓库根目录执行。

## 1. 前置检查

确认工作区状态和当前版本：

```powershell
git status --short --branch
Get-Content package.json
Get-Content src-tauri\tauri.conf.json
Get-Content src-tauri\Cargo.toml | Select-Object -First 8
```

如果工作区有未提交改动，打包产物会包含这些改动。测试包可以不提交；准备正式发布时，建议先确认改动范围再提交。

## 2. 更新版本号

项目使用标准 semver，例如 `1.0.6`。不要使用 `1.06`、`1.0` 这类容易被 npm/Cargo/Tauri 解析异常的写法。

更新 npm 版本：

```powershell
npm.cmd version 1.0.6 --no-git-tag-version
```

同步 Tauri 和 Cargo 版本字段：

```powershell
$version = '1.0.6'
$enc = New-Object System.Text.UTF8Encoding($false)

$p = (Resolve-Path -LiteralPath 'src-tauri\tauri.conf.json').Path
$s = [System.IO.File]::ReadAllText($p)
$s = [regex]::Replace($s, '"version": "[0-9]+\.[0-9]+\.[0-9]+"', '"version": "' + $version + '"')
[System.IO.File]::WriteAllText($p, $s, $enc)

$p = (Resolve-Path -LiteralPath 'src-tauri\Cargo.toml').Path
$s = [System.IO.File]::ReadAllText($p)
$s = [regex]::Replace($s, 'version = "[0-9]+\.[0-9]+\.[0-9]+"', 'version = "' + $version + '"')
[System.IO.File]::WriteAllText($p, $s, $enc)
```

构建 Rust 后，`src-tauri/Cargo.lock` 里的 `app` 包版本会自动同步。

## 3. Windows 测试包

生成 Windows 便携 exe：

```powershell
npm.cmd run package:win:portable
```

成功后会生成：

```text
release/PocPet-<version>-windows-portable.exe
```

为了给测试人员一个短文件名，再复制一份简名：

```powershell
$version = (Get-Content package.json -Raw | ConvertFrom-Json).version
Copy-Item -LiteralPath "release\PocPet-$version-windows-portable.exe" -Destination "release\PocPet-$version.exe" -Force
```

## 4. Android arm64 测试包

先刷新 Tauri Android arm64 原生库：

```powershell
npm.cmd run tauri:android:build
```

在当前 Windows 环境里，这一步通常会在创建 `.so` 符号链接时失败：

```text
Creation symbolic link is not allowed for this system.
```

只要日志里已经出现类似下面的内容，就说明 arm64 Rust 库已经编译完成，可以继续下一步：

```text
Compiling app v<version>
Finished `release` profile
```

然后生成、对齐、debug 签名并校验 APK：

```powershell
npm.cmd run package:android:arm64
```

成功后会生成：

```text
release/PocPet-<version>-arm64-release-debugsigned.apk
```

脚本会输出 Android 版本号：

```text
Android versionName: <version>
Android versionCode: <major><minor><patch padded>
```

例如 `1.0.6` 对应 `versionCode=10006`。

为了给测试人员一个短文件名，再复制一份简名并清理 `.idsig`：

```powershell
$version = (Get-Content package.json -Raw | ConvertFrom-Json).version
Copy-Item -LiteralPath "release\PocPet-$version-arm64-release-debugsigned.apk" -Destination "release\PocPet-$version.apk" -Force
Remove-Item -LiteralPath "release\PocPet-$version-arm64-release-debugsigned.apk.idsig" -Force -ErrorAction SilentlyContinue
```

## 5. 产物核对

列出当前版本产物：

```powershell
$version = (Get-Content package.json -Raw | ConvertFrom-Json).version
Get-ChildItem -Force -LiteralPath release |
  Where-Object { $_.Name -like "PocPet-$version*" } |
  Select-Object Name,Length,LastWriteTime
```

测试分发时优先使用：

```text
release/PocPet-<version>.exe
release/PocPet-<version>.apk
```

详细构建产物也保留：

```text
release/PocPet-<version>-windows-portable.exe
release/PocPet-<version>-arm64-release-debugsigned.apk
```

## 6. 提交顺序

推荐顺序：

1. 更新版本号。
2. 先打包 exe 和 APK。
3. 确认产物能安装/启动。
4. 再提交版本号和相关源码改动。
5. 需要同步远端时再 `git push`。

只提交版本文件时通常包括：

```text
package.json
package-lock.json
src-tauri/Cargo.toml
src-tauri/Cargo.lock
src-tauri/tauri.conf.json
```

打包产物位于 `release/`，已被 `.gitignore` 忽略，不应提交到仓库。

## 7. 常见问题

### Tauri Android symlink 失败

这是 Windows 权限限制，不代表 Rust arm64 库一定失败。确认库已经编译完成后，继续运行：

```powershell
npm.cmd run package:android:arm64
```

### APK 签名

当前测试包使用 Android debug keystore 签名，适合测试分发，不是商店正式签名包。

### Gitee 自动构建

`.gitee/workflows/release.yml` 是自动构建模板。macOS/Linux 产物必须在对应系统 runner 上构建；Windows 本机脚本只负责 Windows exe 和 Android arm64 测试 APK。
