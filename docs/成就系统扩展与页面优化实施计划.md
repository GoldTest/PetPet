# 成就系统扩展与页面优化实施计划

## 目标

- 新增 7 项种植园成就，并为旧存档尽可能回填可确认的历史进度。
- 将回归成就改为回归奖励与累计活跃天数共同贡献的“陪伴印记”。
- 新增类似额外小心心的成就额外产物概率，并接入种植掉落结算。
- 简化成就页信息层级，增加一键领取当前全部可领取奖励。

## 范围与非目标

- 修改成就定义、成就计数、种植掉落结算、成就页面和中英文文案。
- 不修改回归任务 36 小时触发条件、基础种植数值、Mod schema、顶层存档版本或打包版本。
- 不回退当前工作区内正在进行的首页、背包和种植园重构。

## 实施阶段

1. 扩展成就类型、计数归一化、旧存档回填和批量领取接口。
2. 在成功种植、浇水和收获路径记录成就进度，并接入额外产物概率。
3. 增加种植园成就、陪伴印记规则、奖励文本和中英文翻译。
4. 将成就页调整为紧凑列表，增加种植园分类和一键领取。
5. 运行构建并验证奖励、概率边界、旧存档和响应式布局。

## 奖励与规则

- 初次播种：成功种植 1 次，50 金币。
- 第一篮收获：累计收获 1 次，80 金币。
- 勤快浇水：成功浇水 20 次，100 金币、爱心肥料 x1。
- 大丰收：累计收获 30 次，200 金币、爱心肥料 x1。
- 园艺大师：五类树各收获至少 1 次，300 金币、爱心肥料 x2、额外产物概率 +10%。
- 花园之主：解锁全部 5 个栏位，500 金币、额外产物概率 +20%。
- 工具专家：三件种植工具全部达到 3 级，300 金币、额外产物概率 +10%。
- 陪伴印记等于回归奖励领取次数加累计活跃天数除以 30 的整数部分，目标仍为 1、3、10 点。
- 额外产物概率超过 100% 时，整数部分转为必定额外产物，余数再进行一次确定性概率判定。

## 兼容与数据风险

- 新计数字段使用默认值和上限归一化；旧存档从累计收获、当前地块、当日浇水和地块收获次数回填可确认进度。
- 无法证明的历史种植、浇水或已清理树种不做推测，升级后继续累计。
- 已解锁成就不重锁，原有回归成就 ID 和一次性奖励保持不变。
- 一键领取只领取点击时已可领取的奖励，不递归领取领取过程中新解锁的奖励。

## 验证计划

- 运行 `npm run build`。
- 验证成功与失败的种植操作是否正确计数。
- 验证陪伴印记在 30、90、300 活跃日以及混合回归次数下的边界。
- 验证额外产物概率在 0%、99%、100%、120% 以及肥料、天气、营养剂、增益卡组合下的结果。
- 验证旧存档归一化、单项领取、一键领取和重复领取。
- 检查 `1440x900` 和 `390x844` 下的成就列表、分类栏和操作按钮。

## 假设

- 本次不引入新的运行时或测试依赖。
- 当前工作区未提交改动均视为用户工作，实施时只做增量修改。
