# 首页、背包与种植园优化实施计划

## 目标

- 保持现有玩法、存档、Mod 与多语言兼容。
- 强化宠物场景的首页视觉中心地位。
- 将背包改为独立弹层，并提供清晰的分类、使用和跳转操作。
- 收拢种植园的树苗选择与工具升级，减少页面纵向操作密度。
- 统一首页、弹层、种植园及现有功能页的基础视觉语言。

## 范围与非目标

本次调整首页、背包、种植园、通用弹层和样式组织，并拆分过长的页面状态与组件。成就、商店、设置和增益卡只接入统一视觉与弹层行为，不改变业务流程。

本次不修改 `PetState`、存档版本、Mod schema、道具 ID、奖励规则、种植数值和打包版本，也不新增运行时依赖或测试依赖。

## 实施阶段

1. 建立通用弹层外壳、互斥页面/弹层状态与基础视觉变量。
2. 新增背包弹层，支持四类筛选、连续使用、商店切换和种植园跳转。
3. 重排首页信息层级，将升级、天气、季节整合进宠物场景信息区。
4. 将树苗选择和工具升级收进种植园操作弹层。
5. 拆分样式入口，统一重点页面及现有弹层的基础样式。

## 兼容与数据风险

- 所有操作继续调用现有 core 层函数，不复制或改写业务规则。
- 背包分类只依据现有 `ShopCategory`，未知道具继续采用当前兜底定义。
- 弹层切换仅改变 UI 状态，不持久化新的页面状态。
- 种植园返回商店时保留当前地块，商店关闭后继续停留在种植园。

## 验证计划

- 运行 `npm run build`，验证 TypeScript 与生产构建。
- 在 `1440x900` 和 `390x844` 视口检查首页、背包和种植园。
- 验证背包分类、空状态、连续使用、商店切换和种植园跳转。
- 验证树苗选择、工具升级、直接养护、收获和清理确认流程。
- 检查弹层 `Escape` 关闭、焦点恢复、滚动限制和固定操作栏遮挡。

## 已确认假设

- 桌面端与移动端同等优先。
- 保留现有宠物和道具素材，进行中度视觉统一。
- 首页以宠物场景为首要视觉重点。
- 不引入自动化测试框架，使用构建检查与一次聚焦的 Browser 双端验证。
