# PocPet 开发路线图

本文记录 PocPet 后续开发路线。这里的版本是规划版本，不等同于当前正式发版号；正式版本号以 `package.json`、Tauri 和 Cargo 配置为准。

## 设计原则

- 优先做低压力长期陪伴，不做高压签到、失败惩罚或强任务清单。
- 每天打开应有轻微变化，离开几天不应被重罚。
- 长期游玩要留下个人痕迹，奖励优先服务陪伴感、收集感和记忆感。
- 新玩法保持短操作、低打扰，适合桌面和移动端碎片时间。
- 大型扩展和基础养成存档解耦，避免破坏旧存档兼容。

## 1.0.x：当前基础版本

定位：稳定基础养成、商店、背包、番茄钟、Mod v1 和跨平台测试包。

已包含：

- 基础宠物状态：饥饿、清洁、心情、精力、健康、等级、睡眠。
- 照顾操作：喂食、清洁、玩耍、工作、触摸、使用道具。
- 经济系统：金币、小心心、商店、背包、每日饼干、每日折扣、小心心换金币。
- 番茄钟：专注 / 休息流程、活动反馈、专注统计。
- 日期奖励：生日、周年、节日、月度礼物、年度统计和年度总结。
- Mod v1：宠物图片、道具图片、默认名字、默认生日、文本和喜欢食物。
- 存档导入导出：`PocPetSaveFileV1`、裸 `PetState` 兼容、时间基线重置。
- Windows exe 和 Android arm64 测试包流程。

维护重点：

- 修复打包、Android 安全区、资源映射、旧存档补齐和 Mod 导入问题。
- 保持当前轻量体验，不提前引入重系统。

## 1.1：长期陪伴基础阶段

定位：下一阶段 / 候选版本，用当前已实现内容和低压力长期目标串起每日回访理由。此章节不表示当前正式版本已经是 1.1。

已完成或优先纳入：

- 每日小愿望：每天 6 点刷新，目标池包含喂食、清洁、玩耍、摸摸和打工一次。
- 回归欢迎：离开 36 小时以上触发恢复任务和轻量补给，不做强惩罚。
- 纪念册 / 回忆册设计：记录生日、节日、年度总结、第一次升级、第一次获得心、番茄钟累计、陪伴天数等长期节点。
- 成就系统第一版设计：作为“长期里程碑 + 轻量目标反馈”，记录条件、进度、完成状态和轻量奖励。

成就系统方向：

- 分类：陪伴、照顾、成长、番茄钟、收集、节日、隐藏成就。
- 奖励：普通成就给少量金币、小心心或低价值道具；重要成就优先给称号、贴纸、纪念卡、头像框、饰品或小窝装饰。
- UI：未来放在“纪念 / 成就”页面，用 Tab 区分故事记录和进度目标。
- 数据：后续新增 `achievements` 状态；跨年度累计需要 lifetime stats，不能只依赖 `yearlyStats`。

非目标：

- 不做强制连续签到成就。
- 不做失败惩罚成就。
- 不让成就奖励破坏金币、小心心和道具经济。

## 1.2：中期增强阶段

定位：补足长期消耗点、外观收集和温和回访理由。

计划内容：

- 小窝 / 房间装扮：用金币解锁背景、家具、地毯、摆件和玩具；第一版先支持背景和少量摆件，不做复杂摆放编辑器。
- 宠物饰品：围巾、铃铛、帽子、睡帽、节日装饰等永久外观，可通过商店、节日、生日、年度纪念或特殊成就获得。
- 种植 / 花园：新增独立花园页面，用金币种普通树苗，少量稀有树苗可消耗小心心；树苗随真实时间成长，成熟后产出橘子、苹果、香蕉等现有水果道具。
- 宠物偏好扩展：基于食物、互动、作息或番茄钟活动形成长期偏好，主要影响反馈文本、每日小愿望和轻量奖励。

种植边界：

- 第一版采用轻照顾设计，浇水或照顾可加速或增产。
- 忘记照顾不会枯死，不惩罚离开几天的玩家。
- 每棵树用剩余可收获次数表示寿命，次数用完后进入耗尽状态，可清理释放种植位。
- 不把种植做成强制每日打卡，不让小心心树苗抢占升级系统核心消耗。

## 1.x 待定：长期差异化阶段

定位：增强不同玩家存档的差异和宠物个性，具体版本暂时留空。

候选内容：

- 性格标签：根据喂食、清洁、番茄钟、摸摸、礼物等行为形成安静、黏人、努力、贪吃、爱干净等倾向。
- 记忆文本：宠物能提起近期或长期行为，例如最近常陪它读书、这个月完成很多番茄钟、常在晚上来看它。
- 季节活动扩展：春夏秋冬限定小事件、限定装饰和限定纪念物。

边界：

- 性格不做复杂天赋树，第一版可先用隐藏分数和显示标签。
- 记忆文本基于本地统计生成，不依赖联网服务。
- 季节活动不要求每天强刷，适合低频收集和节气氛围。

## 2.0：冒险派遣 / 冒险 DLC

定位：大型扩展内容，作为独立玩法层推进，不污染基础养成存档。

核心原则：

- `PetState` 继续作为宠物身份、等级、属性、金币、小心心和背包的基础来源。
- 冒险数据独立保存，建议本地键为 `pocpet.adventure.v1`。
- 导出时可通过可选 `extensions.adventure` 携带冒险数据。
- 旧基础存档、裸 `PetState`、当前 `PocPetSaveFileV1` 都应能进入冒险模式并生成默认冒险状态。
- 小心心不进入冒险商店，继续保留给基础升级系统。

第一版玩法方向：

- 采用派遣式玩法，不做实时战斗或复杂回合制。
- 玩家选择地点、时长和装备后开始派遣。
- 派遣中显示倒计时、地点、天气影响和简单动画。
- 到时后生成战报、奖励和轻微状态变化。
- 可支持提前召回，但只给少量或不给奖励，并减少状态损耗。

建议内容：

- 冒险材料、装备背包、装备槽位、每日冒险商店、当前派遣、最近战报。
- 地点先做三档：15 分钟、45 分钟、90 分钟。
- 派遣限制覆盖睡眠、番茄钟运行、低健康、低体力。
- 结算使用随机种子固定，避免刷新刷奖励。

非目标：

- 不做服务器存档。
- 不做真实平台 DLC 付费校验。
- 第一版不做实时战斗、复杂技能树或多人内容。
- 第一版不改变当前宠物升级消耗公式。

## 未来待定

- 更复杂的收集图鉴：装备、材料、区域、纪念物和事件图鉴。
- 更完整的 Mod 扩展：装扮、背景、节日奖励池或冒险扩展内容。
- 更完整的房间编辑器、外观奖励和成就联动装扮。

这些内容需要在基础陪伴体验、存档兼容和资源规范稳定后推进。
