# PocPet 1.2 开发计划

## 目标

1.2 的目标是补齐长期收集目标，同时保持 PocPet 轻量陪伴的节奏：新增独立成就页、第一批成就与少量稀有成就奖励，并把“西瓜”加入现有水果道具体系。实现时优先保护旧存档、现有 Mod 和当前经济节奏。

## 范围

- 新增水果道具：西瓜。
- 新增独立成就页面：主界面点击奖杯图标进入，页面内展示成就进度、已达成状态、奖励说明和稀有标记。
- 新增成就系统第一版：覆盖照顾、道具、番茄钟、回归、年度统计、节日/生日等现有行为；详细列表见 `docs/1.2成就列表设计.md`。
- 新增稀有成就奖励：少量永久增益或轻量收入，奖励需要可解释、可叠加、可限制上限。
- 保留现有日常愿望、回归欢迎、年度回顾、节日奖励、商店、背包和 Mod 逻辑。

## 非目标

- 不做连续签到惩罚，也不把成就做成每日任务清单。
- 不引入联网排行、账号、云存档或付费能力。
- 不让稀有成就奖励显著破坏金币、爱心和商店价格平衡。
- 不要求 Mod 在 1.2 立刻支持自定义成就逻辑；最多允许覆盖新增西瓜图标资源。
- 不在本阶段强制完成小窝、饰品、花园和纪念册的完整系统；这些可以作为后续 1.2.x 或 1.3 拆分。

## 当前代码基础判断

- 道具体系集中在 `src/core/petTypes.ts`、`src/core/items.ts`、`src/assets.ts`、`src/i18n/*.json` 和 Mod 资源白名单。
- 存档通过 `normalizePet` 容错补字段，没有显式 `PetState` schemaVersion；新增字段应提供默认值并允许旧存档自动补齐。
- 主界面目前使用弹窗和面板组合，商店/设置通过顶部图标或按钮进入；成就页适合作为 App 内的全屏/准全屏页面状态，而不是塞进设置页。
- 年度统计、日常愿望、回归欢迎和番茄钟已记录了部分长期行为，可作为成就进度来源，但仍缺少部分累计计数。

## 数据设计

### 西瓜道具

- `ItemId` 增加 `watermelon`。
- 类型：`food`。
- 建议价格：`26`。
- 建议效果：`hunger +28`、`mood +2`、`cleanliness -2`、`health +1`。
- 加入水果池：月初礼物、节日水果池、收获/随机食物池等使用 `fruitItems` 的位置。
- 图标资源：新增 `src/assets/icon/item_watermelon.png`，并允许 Mod 使用 `watermelon.png` 覆盖。
- 中英文文案补齐：名称和摘要。

### 成就存档字段

建议新增：

```ts
export type AchievementId = string;

export interface AchievementState {
  unlockedAtById: Partial<Record<AchievementId, number>>;
  claimedRewardIds: AchievementId[];
  counters: AchievementCounters;
}

export interface AchievementCounters {
  careActionCounts: Record<YearlyCareActionKey, number>;
  totalItemUseCount: number;
  totalPomodoroFocusCount: number;
  totalHeartEarned: number;
  totalCoinsEarned: number;
  rareIncomeClaimDateKey?: string;
}
```

兼容策略：

- `normalizePet` 对缺失 `achievements` 自动补默认结构。
- 对旧存档可用现有 `yearlyStats` 回填一部分初始进度，但不 retroactive 解锁所有旧行为，避免玩家打开旧存档瞬间刷屏。
- 成就奖励是否领取独立于“已达成”，避免未来新增奖励时重复发放。

### 成就定义

建议新增 `src/core/achievements.ts`：

- 集中定义 `AchievementDefinition`。
- 提供 `evaluateAchievements(pet, event, now)`，在行为发生后解锁。
- 提供 `getAchievementProgress(pet, id)`，供 UI 展示进度。
- 提供 `getAchievementEffects(pet)`，统一计算永久增益。

示例字段：

```ts
interface AchievementDefinition {
  id: AchievementId;
  rarity: 'normal' | 'rare';
  category: 'care' | 'collection' | 'pomodoro' | 'milestone' | 'season';
  titleKey: string;
  descriptionKey: string;
  progressTarget?: number;
  reward?: AchievementReward;
}
```

## 稀有成就奖励原则

稀有奖励只给“轻量、上限明确、长期可感知”的收益。

推荐奖励类型：

- `coinIncomeDaily`：每天进入成就页或首次打开 App 时可领取少量金币，例如 +4 或 +6。
- `workCoinBonus`：打工基础金币 +1，上限累计不超过 +3。
- `pomodoroCoinBonus`：番茄钟专注奖励 +1，上限累计不超过 +3。
- `shopDiscountTiny`：每日折扣之外的固定小折扣，例如指定分类 -1 金币，不和免费道具叠加。
- `careStatBonus`：特定行为额外 +1 状态值，只影响舒适感，不直接刷金币。

不推荐奖励：

- 直接大额金币或爱心。
- 无限倍增收入。
- 降低所有价格的百分比折扣。
- 连续登录相关强压力奖励。

第一批稀有成就建议：

- `deep_focus`: 累计完成 100 次番茄钟专注，奖励番茄钟金币 +1。
- `trusted_companion`: 陪伴满 30 天且活跃 10 天，奖励每日成就津贴 +4 金币。
- `gentle_caretaker`: 清洁、喂食、玩耍、摸摸各累计 50 次，奖励对应照顾行为额外 +1 心情或健康。
- `fruit_collector`: 使用橙子、苹果、香蕉、西瓜各 3 次，奖励每日成就津贴 +3 金币。

## 页面与入口

- 顶部工具区新增奖杯图标按钮，使用 `lucide-react` 的 `Trophy`。
- 点击后进入独立成就页，建议用 `activePage: 'home' | 'achievements'` 控制。
- 成就页应有返回按钮、分类筛选、整体进度、稀有成就区域、普通成就列表。
- 已解锁成就显示达成时间；未解锁成就显示进度，隐藏型稀有成就可只显示提示。
- 稀有增益在成就页明确显示“已生效”或“可领取”，避免玩家不知道收益来源。
- 移动端底部操作区保留原有照顾按钮；进入成就页时隐藏宠物主界面的大块交互，避免叠层拥挤。

## 实施阶段

### 阶段 1：西瓜道具

- 更新 `ItemId`、商店道具、资源映射、i18n 文案。
- 加入 `dateRewards` 相关水果池。
- 更新中英文 Mod 制作指南的可覆盖文件名列表。
- 准备 `item_watermelon.png`，尺寸和风格跟现有水果图标一致。

### 阶段 2：成就核心

- 新增成就类型、默认状态、normalize 兼容逻辑。
- 在 `applyPetAction`、`useInventoryItem`、`advancePomodoro`、`upgradePet`、日期奖励领取等行为后记录 counters 或触发评估。
- 控制解锁提示：每次行为最多弹出少量成就，避免多个历史成就一次性打断。

### 阶段 3：奖励与增益

- 实现 `getAchievementEffects`。
- 接入打工收益、番茄钟收益、每日成就津贴和轻量状态奖励。
- 对所有收入类奖励设定硬上限，并在 UI 展示来源。
- 给稀有奖励加 `claimedRewardIds` 或日期字段，避免重复领取。

### 阶段 4：成就页 UI

- 新增 `AchievementPage.tsx` 或 `AchievementsPage.tsx`。
- 顶部增加奖杯入口，成就页内提供返回。
- 增加成就卡片、进度条、稀有标记、奖励说明。
- 补齐 `zh-CN` 和 `en-US` 文案。
- 检查桌面和移动端布局，避免按钮和文字溢出。

### 阶段 5：验证与平衡

- 跑 `npm.cmd run build`。
- 手工验证旧存档加载、新存档创建、导入导出、购买/使用西瓜、月初礼物水果池、成就解锁、稀有奖励不重复发放。
- 重点检查经济：稀有奖励全开后，每天额外收入建议控制在 10-20 金币以内，单次行为收益增益不超过 +3。

## 存档与兼容风险

- 新增 `watermelon` 不会破坏旧存档；旧存档没有该物品时默认为 0。
- Mod 资源白名单新增 `watermelon.png` 后，旧 Mod 不受影响；新 Mod 可选择提供图标。
- 成就 counters 如果只从 1.2 开始累计，会让老玩家部分历史不被承认；如果完全回填，可能出现启动后大量解锁。建议只回填低风险总量，例如当前年度统计，并限制首次启动提示数量。
- 稀有成就收益需要集中计算，避免在多个函数里重复加成。
- 导入旧版本存档时要 normalize 成就字段，导出存档继续使用现有 save schema，除非后续确认需要升级 save schemaVersion。

## 测试计划

- 新存档：确认 `achievements` 默认值存在，首页奖杯入口正常。
- 旧存档：确认缺失 `achievements`、`watermelon` 时可加载并保存。
- 商店：购买西瓜、金币扣除、背包数量、使用效果和音效符合预期。
- 日期奖励：月初礼物和水果类节日可能发放西瓜。
- 成就触发：照顾、使用水果、完成番茄钟、升级、回归任务后能更新进度。
- 稀有奖励：收入类每日只领取一次，永久增益不会重复叠加。
- UI：桌面和移动宽度下成就页列表、进度条、奖励说明不溢出。
- 导入导出：带成就字段的存档可导出再导入，旧存档导入不报错。

## 开放问题

- 西瓜是否需要直接进商店，还是先只通过奖励/节日/花园获得。当前建议直接进商店，降低验证复杂度。
- 稀有成就是自动领奖，还是在成就页点击领取。当前建议永久增益自动生效，金币津贴在成就页领取，减少重复发放风险。
- 第一版是否需要隐藏成就。当前建议最多 1-2 个隐藏稀有成就，其余全部可见。
- 是否把“纪念册”和“成就”合并入口。当前建议 1.2 先做成就独立页，纪念册后续单独规划。
